Scalability & Future Plan:
This document outlines how the current project solution can be scaled to handle larger user bases, increased data loads, or extended features in the future. It also captures the team's roadmap for enhancing and evolving the project beyond its current state.
Current System Limitations:
S.No Limitation Impact Priority to Address (High / Medium / Low)
1 SQLite single-file database. Write locks occur during simultaneous bulk CSV uploads. High
2 In-process model inference. Blocks the main Flask thread under heavy API request volume. Medium
3 Client-side jsPDF rendering. Consumes client-side memory and could fail on large historical exports. Low
Scalability Plan:
S.No Scalability Aspect Current State Proposed Upgrade / Solution
1 User Load Flask built-in server (single worker process). Migrate to Gunicorn or Nginx with multi-threaded worker configurations.
2 Data Storage Single-file SQLite database. Migrate database to PostgreSQL or MySQL for concurrent operations.
3 Performance Synchronous predictions. Integrate Celery task queues with Redis to handle batch evaluations asynchronously.
4 Security Cookie sessions. Upgrade to JSON Web Tokens (JWT) for stateless session validation.
Future Roadmap:

Phase 2 Database Scale & API Migration Month 3 Seamless handling of multiple bulk uploads.
